# ai-draw 小白使用说明

你可以把 ai-draw 理解成：

```text
Codex 里的 AI 画图白板。
```

第一次使用前先安装插件。以后每次用，只要在 Codex 里说你想做什么图，再去画布上标注修改意见。

## 1. 第一次安装

如果你是从 GitHub 下载：

```bash
git clone https://github.com/nan1888/ai-draw.git
cd ai-draw/ai-draw
npm run setup
cd ..
codex plugin marketplace add .
codex plugin add ai-draw@ai-draw
```

然后重启 Codex，或新建一个对话。

完整安装说明见：

```text
INSTALL.md
```

## 2. 怎么判断安装好了

在 Codex 里输入：

```text
@ai-draw 打开 AI 画布。
```

如果成功，Codex 会返回一个本地画布网址，类似：

```text
http://127.0.0.1:43218/
```

点开这个链接，应该能看到画布。

如果 Codex 说找不到 ai-draw、找不到 MCP 工具，通常是插件还没重新加载。请完全退出并重新打开 Codex，或新开一个对话。

## 3. 日常使用流程

### 第一步：打开画布并生成图片

在 Codex 里输入：

```text
@ai-draw 打开 AI 画布，帮我做一张拉面广告。
```

也可以更具体：

```text
@ai-draw 生成一张竖版拉面广告，品牌叫拉面一番，要高级食物摄影风格。
```

Codex 会自动准备画布、创建图片框、生成图片，并把图片放进画布。

### 第二步：打开画布链接

Codex 会返回类似这样的链接：

```text
http://127.0.0.1:43218/
```

直接在 Codex 里点开，让画布在侧边栏显示。

### 第三步：在画布上标注

如果图片哪里不满意，你可以在画布上：

1. 画箭头。
2. 圈出区域。
3. 写文字说明。

推荐写法：

```text
这里用白汤
碗小一点
背景更干净
右上角留 logo 区域
```

不推荐：

```text
改一下
这里不好
感觉不对
```

### 第四步：开启自动修图模式

第一次改图前，在 Codex 里说：

```text
@ai-draw 开启自动修图模式
```

之后你每次在画布标完一批，就点击画布上的：

```text
按标注修图
```

Codex 会读取这批标注，生成新版图片，并放到旧图右侧。

## 4. 需要记住的三句话

生成图片：

```text
@ai-draw 打开 AI 画布，帮我做一张拉面广告。
```

开启监听：

```text
@ai-draw 开启自动修图模式
```

提交修改：

```text
在画布上标注，然后点“按标注修图”。
```

## 5. 常见问题

### Codex 找不到 ai-draw

处理方式：

1. 确认已经安装插件。
2. 重启 Codex。
3. 新开一个对话。
4. 再输入 `@ai-draw 打开 AI 画布。`

### 画布网页没打开

如果 Codex 返回了网址，可以手动点开：

```text
http://127.0.0.1:43218/
```

如果端口被占用，可以让开发者设置：

```bash
export AI_DRAW_PORT=43219
```

### 图片没有插入画布

对 Codex 说：

```text
把刚才生成的图片放进画布里。
```

### 点了按钮但没有修图

先确认 Codex 正在自动监听。回到 Codex 说：

```text
@ai-draw 继续自动修图
```

然后再点画布上的 `按标注修图`。

## 6. 普通用户不需要知道什么

正常使用时，你不需要知道：

1. MCP 是什么。
2. holder 是什么。
3. run metadata 是什么。
4. assets 保存在哪里。
5. `.mcp.json` 怎么写。
6. 服务怎么启动。

这些都应该由插件和 Codex 自动处理。

## 7. 开发者补充

安装和构建：

```bash
npm run setup
```

检查项目：

```bash
npm run typecheck
npm run test
npm run validate:plugin
```

手动启动画布服务：

```bash
NODE_ENV=production node packages/canvas-app/dist/server/server.js \
  --port 43218 \
  --workspace-root "<your workspace>"
```

本地画布数据默认保存在当前工作区的 `.ai-draw/` 目录。这个目录不会提交到 Git。
