# ai-draw 自然语言工作流

这份文档只讲普通用户怎么用。

## 一句话版本

先安装 ai-draw 插件。  
安装后，在 Codex 里说：

```text
@ai-draw 打开 AI 画布，帮我做一张拉面广告。
```

然后在 Codex 开启自动修图模式：

```text
@ai-draw 开启自动修图模式
```

之后在画布上标完一批，点右侧 `按标注修图`，Codex 会自动接任务。

## 第一次使用

第一次使用要先安装插件。

理想情况下，用户应该这样做：

1. 打开 Codex。
2. 进入插件/扩展页面。
3. 安装 `ai-draw`。
4. 重启 Codex，或者新开一个对话。
5. 输入：

```text
@ai-draw 打开 AI 画布，帮我做一张图。
```

如果你的 Codex 使用 `/` 调用插件，那就输入：

```text
/ai-draw 打开 AI 画布，帮我做一张图。
```

如果 Codex 会自动识别插件，也可以直接说：

```text
打开 AI 画布，帮我做一张图。
```

## 每次做图

你可以直接说：

```text
@ai-draw 帮我做一张小红书封面，主题是春季咖啡新品。
```

或者：

```text
@ai-draw 生成一张竖版拉面广告，品牌叫拉面一番，要高级食物摄影风格。
```

Codex 应该自动：

1. 打开画布。
2. 创建图片区域。
3. 生成图片。
4. 把图片放进画布。

画布地址会作为本地链接显示在 Codex 对话里。用户点击链接，在 Codex 侧边栏操作画布；不要单独弹出外部浏览器窗口。

## 每次改图

先在画布上标注：

1. 用箭头指向要改的地方。
2. 在箭头旁边写修改意见。
3. 可以用圆圈或矩形圈出区域。

第一次改图前回到 Codex 说：

```text
@ai-draw 开启自动修图模式
```

Codex 应该自动：

1. 监听画布按钮提交的修图任务。
2. 读取当前画布。
3. 理解箭头、文字和圈选。
4. 修改图片。
5. 把新版放到旧版右边。
6. 保留旧图。

之后每次改图，只需要在画布上标完一批并点击：

```text
按标注修图
```

如果画布提示 `Codex 已暂停`，说明监听空闲太久已经停了。回到 Codex 说：

```text
ai-draw 继续自动修图
```

Codex 会接着处理画布里已经提交的任务。

## 推荐你这样说

生成拉面广告：

```text
@ai-draw 打开 AI 画布，生成一张竖版拉面品牌广告，品牌名叫拉面一番，风格高级、温暖、有食欲。
```

生成小红书封面：

```text
@ai-draw 帮我做一张小红书封面，主题是春季咖啡新品，画面清新、明亮、适合种草。
```

根据标注修改：

```text
@ai-draw 开启自动修图模式
```

继续修改：

```text
在画布上继续标注，然后点“按标注修图”。
```

## 标注怎么写更好

推荐：

```text
这里用白汤
碗小一点
背景更干净
右上角留 logo 区域
整体更暖一点
去掉这行文字
```

不推荐：

```text
改一下
这里不好
感觉不对
```

因为太短、太模糊，Codex 不一定知道你要改什么。

## 普通用户不需要做的事

你不需要：

1. 手动调用 MCP 工具。
2. 手动创建 holder。
3. 手动复制图片路径。
4. 手动保存 run 文件。
5. 理解 `.mcp.json`。

如果文档里出现这些词，那是给开发者看的，不是给普通用户看的。
